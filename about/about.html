Reason for Creation

As mentioned in the header of the Javascript file (and in the footer) this project was completed for a Media Arts and Design (MAAD) course at the University of Chicago. This course is named Internet Art II and focuses on the creative possibilites enabled through the browser and web.

For this project, we focused on making art with some kind of text generation.

Inspiration

Given the constraint of text generation, I thought hard for a project that would incorporate this. One thing we covered in this class was how randomness was a common theme in computer generated art, and it was this that got down the path of this project.

I remebered the classic ransom note from TV / Movies (see Fig. 1 ). Although the text was deliberate, the crazed styling of them seemed like they could be modeled well by using randomness. Further, the variation in sizing, colors, and spacing seemed like it would be an entertaining project to work on!

Methodology

I documented the pipeline of how the characters are styled well (at least to my standards) in the Javascript which is avaliable either on my GitHub or viewable in the dev tools here directly. Thus I will not go into depth. Instead I will use this section to document the reason behind my design choices.

To achieve the random look of the project, I varied 5 components - text size, color, background color, tilt of the character, and spacing between them. For all but spacing I used a uniform distribution - taken by scaling and shifting the Math.random() function - but for spacing I used a gaussian distribution.

The uniform distribution gives a varied and random appearance to the style. From looking at example notes, I felt they all had this property - a hint at the deranged nature of the adversaries who wrote the note (not that the users of this project are like that though!). However, I wanted to use a gaussian for the spacing between the charactes as I felt this would model human nature better. I thought people would generally place them in the same spacing, but might vary a little bit when placing them. This lends itself well to a gaussian distribution. I also just wanted to experiment with it and found a cool formula to generate a gaussian distribution using Math.random() twice.
